项目名称：cyber backup

项目介绍introduction

人总想留下些什么。

有人放不下一段感情。
有人翻遍聊天记录，只想再看一句“到家了吗”。
也有人终于想起回头时，那个会发六十秒语音的人，已经不在了。

可人的声音会散。
说话的习惯会忘。
连那些最平常的停顿、语气与碎碎念，也会慢慢沉进时间里。

赛博备份，是一个半自动的人格化聊天系统。

它会监听输入、记录上下文、维持记忆，并调用 AI 自动生成回复。
系统拥有剪贴板监听、人格记忆、防回流、聊天节奏拆分、自动发送等功能。

它不会像普通机器人一样，一次吐出完整而冰冷的大段文字。

它会模仿真实聊天里的停顿。

系统会逐渐学习一个人的说话习惯。
短句、语气、口头禅、停顿方式，都会缓慢沉积进记忆。

你说一句。
它记一句。

它或许无法真正留下一个人。

但很多年后，
当聊天框再次亮起时，
至少那种熟悉的说话方式，还会像从前一样慢慢出现。

像回音。
也像未散的旧梦。

常见问题

1. 为什么会出现“自问自答”？

这是因为机器人生成的回复被再次识别成了用户输入，重新写入了 inbox。

项目早期曾多次出现回流问题。
后续通过：

  • _bot_ 标记
  • 剪贴板过滤
  • AHK 发送前清洗
  • 输入防抖

等方式解决。

2. 为什么聊天会一句一句发送？

系统会主动拆分长句。

因为真实聊天并不是一次发送大段文字，而是带有停顿、迟疑与节奏感。

例如：

这种效果会比一次性输出更接近真人聊天。

3. 为什么有时回复会比较慢？

系统存在输入缓冲机制。

当用户连续发送短句时，监听器会等待数秒，直到输入稳定后再统一生成回复。

这是为了避免：

  • 抢话
  • 上下文断裂
  • 一句话触发一次 AI

4. 为什么需要 memory？

memory 并不是为了“长期记住所有内容”。

而是为了：

  • 保持语气统一
  • 维持人格
  • 记录最近聊天状态
  • 模拟连续对话

系统默认只读取最近部分记忆窗口。

5. 为什么不用官方聊天界面，而是自己写监听器？

因为项目目标并不是“聊天”。

而是：

监听器、AHK、memory 与 API 共同组成了一个半自动人格系统。

更新日志及反思

v0.1

最初版本。

只有：

  • 剪贴板监听
  • API 调用
  • 自动发送

能够实现最基础的自动聊天。

但回复机械、AI感强，且容易死循环。

v0.2

加入：

  • persona
  • self
  • decision

系统开始拥有初步人格。

但此时仍存在严重回流问题。

v0.3

加入 memory。

系统开始具备上下文能力。

但也因此出现：

形成自问自答。

这是项目第一次真正意义上的“人格污染”。

v0.4

开始处理回流。

加入：

  • _bot_ 标签
  • 剪贴板过滤
  • AHK 清洗
  • 重复输入检测

系统稳定性明显提高。

也第一次意识到：

v0.5

加入输入缓冲与防抖。

系统不再“见一句回一句”。

而是等待用户输入稳定后，再统一生成回复。

聊天节奏开始接近真人。

v0.6

开始拆分消息发送。
