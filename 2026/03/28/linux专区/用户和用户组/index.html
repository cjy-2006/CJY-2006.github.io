用户和用户组

root用户

在linux中最大权限的用户为root用户（超级管理员）

普通用户的权限一般在home（~）目录内不受限的

在unbunt发行版本中，buntu 系统为了安全，默认情况下root 账户是被锁定的，无法直接通过 su - root 登录。临时切换到 root 环境（推荐）sudo -i

su和exit命令

  1. su：用于账户切换

  2. 语法： su [-] 用户名

    • ‘ -’ 符号是可选的，表示是否在切换用户之后加载环境变量
    • 参数：用户名，也可以省略，省略表示切换到root
    • 切换用户之后，可以用exit返回上一用户

sudo命令

长期切换至root用户会带来系统损坏，为避免此情况，会用sudo命令进行临时授权

  1. sudo为普通命令进行授权，临时以root身份执行

  2. 语法： sudo 其他命令

  3. 配置用户：用visudo加相应配置项cjy ALL=(all) NOPASSWD: ALL

    sudo -i可以进入root权限

用户和用户组

linux系统可以

  • 配置多个用户
  • 配置多个用户组
  • 用户可以加入多个用户组之中

linux权限管控有两个级别

  • 针对用户
  • 针对用户组

用户组管理

以下命令要root权限

创建用户组

groupadd 用户组名

删除用户组

​ groupdel 用户组名

用户管理

创建用户

​ useradd [-g -d -m] 用户名

  1. -g 指定用户的组，不指定-g会创建同名组并加入，如果选择-g指定用户组，则被指定的用户组要存在

  2. -d指定用户home路径，不指定home目录默认存在/home/用户名

  3. -m 是 useradd 的核心参数，全称 --create-home，核心作用是自动创建用户的家目录，并从 /etc/skel 复制 .bashrc 等默认配置文件到该目录；

    不加 -m 时，系统仅在 /etc/passwd 中记录家目录路径，不会实际创建目录，用户登录后无可用家目录；

    -m 常与 -d 配合：-d 指定家目录自定义路径，-m 负责创建该路径，新手创建可登录用户时建议必加 -m。

用户删除

userdel -r 用户名

  1. -r表示删除用户home目录

查看用户所在组

id 用户名

修改用户所属组

usermod -aG用户组 用户名，将用户加入指定用户组

getent命令

getent passwd 查看当前系统中有哪些用户

getent group 查看系统用户组
