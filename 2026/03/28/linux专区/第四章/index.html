软件安装

yum命令

yum：RPM包安装管理器，用于 cent os

语法： yum -y install | remove | search 软件名称

-y选项：是否自动同意

apt命令

用于Ubuntu，区别于cent os Ubuntu使用apt管理器，Ubuntu安装包是.deb

语法和yum一样

语法： apt [-y] install | remove | search 软件名称

systemctl命令

linux中的很多软件可以通过systemctl来控制：启动，停止，开机自启，关闭开机自启，能够被systemctl管理的软件一般被称为：服务

语法;systemctl start | stop | status(查看状态) | enable(开机自启动) |disable（关闭开机自启）

常见服务名称

  • 主网络服务 ：NetworkManager

  • 副网络服务：network

  • 防火墙：ufw（ubunt）

  • sshd，ssh服务（finalshell远程登录）

    第三方软件安装后也可以用systemctl进行控制

软连接

类似于Windows的快捷方式

ln 命令

语法ln -s 参数一 参数二

  • -s选项： 创建软连接，如果不选就是硬链接

    硬链接：同一个文件的多个别名，共用同一个 inode 和数据块。

    软链接：独立的小文件，存放指向目标文件的路径，类似 Windows 快捷方式。

  • 参数一： 被连接的文件和文件夹

  • 要链接去的地方

日期和时区

date命令

语法 date [-d] [+格式化字符串]

  • -d 按照给定字符串显示日期，一般用于日期计算。例如：date -d’+1 day‘ +%Y-%m-%d

​ 其中支持的时间标记为Year Month Day hour Minute second

  • 格式化字符串：通过特定的字符串标记，来控制显示日期
    • %Y:年
    • %y：年份后两位
    • %m：月份
    • %d：日
    • %H：小时
    • %M：分钟
    • %S：秒
    • %s：字1970年1.1日00:00:00UTC到现在的秒数

ip地址

每一台计算机都会有一个地址用于和其他计算机通讯

ip地址版本：v4版本和v6版本

ipv4格式：a.b.c.d,abcd 表示0~255的数字，可以通过ifconfig查看本机地址

特殊ip地址

127.0.0：该地址用于代指本机

0.0.0.0:

  1. 可以用于指代本机 

        2. 可以在端口绑定中用于确定绑定关系
    3. 在一些IP地址的限制中表示所有ip的意思，例如放行规则设置为0.0.0.0表示允许仍和ip地址访问

端口

指设备和外界交互的出入口

公认端口：1–1023，通常用于系统内置程序或知名程序使用，如ssh在22端口，HTTPS在443端口非特殊情况不占用端口

注册端口：1024 ~49151可以随意使用，通常用于松散的绑定一些程序/服务

动态端口：49152 ~65535通常不会邦定程序，在程序对外链接是临时使用

namap ip：查看端口占用情况（ip是指计算机的ip地址）

netsat命令：用于查看制定端口占用情况

进程管理

每一个程序运行时都会被操作系统记录成一个进程，并会为每一个进程分配一个ID（进程号）

ps命令

语法：ps [-e -f ]

  • -e：显示所有进程
  • -f以完全格式化的形式展示信息
  • 固定用法ps -ef一般要和管道符连用

| UID | PID | PPID | C | STIME | TTY |TIME |CMD

  1. UID:运行这个进程的用户（谁启动的）

  2. PID:进程 ID（进程的唯一编号）

  3. PPID:父进程 ID（谁启动了这个进程）

  4. C:CPU 占用率 / 调度信息（

  5. STIME:进程启动时间

  6. TTY:进程运行在哪个终端

  • ? 表示后台系统进程（没有终端）

  1. TIME:进程累计占用 CPU 的时间

  2. CMD:进程启动的命令 / 程序名（最关键）

kill命令

用途：强制结束进程

语法kill -9强制关闭

主机状态

查看系统资源占用

top命令

每五秒刷新一次，形单于Windows的任务管理器

无参数，无选项

字段信息

字段 含义
PID 进程唯一 ID
USER 进程所属用户
PR 进程优先级（值越小优先级越高）
NI nice 值（-20~19，负值优先级更高）
VIRT 虚拟内存总量（KB）
RES 常驻物理内存（未换出，KB）
SHR 共享内存大小（KB）
S 进程状态（R 运行 / S 睡眠 / D 不可中断 / Z 僵尸 / T 停止）
%CPU 进程 CPU 使用率
%MEM 进程物理内存使用率
TIME+ 累计 CPU 时间（1/100 秒）
COMMAND 进程命令 / 程序名

磁盘信息监控

df命令

查看磁盘使用情况
选项：

  • -h ：以更加人性化的单位显示
  • -T：显示文件类型

iostat

语法：iostat -x num1 num2

  • -x：显示更多信息
  • num1：刷新间隔
  • num2：刷新几次

-x显示的信息

简写 信息
rrqm/s 每秒合并的读请求数
wrqm/s 每秒合并的写请求数
r/s 每秒完成的读 I/O 次数
w/s 每秒完成的写 I/O 次数
rMB/s 每秒读取数据量（MB）
wMB/s 每秒写入数据量（MB）
await I/O 请求平均等待时间（ms）
r_await 读请求平均等待时间（ms）
w_await 写请求平均等待时间（ms）
aqu-sz 平均 I/O 队列长度
svctm I/O 平均服务时间（ms）
%util 设备繁忙利用率（关键指标）

环境变量

指操作系统运行时记录的关键性息，已辅助系统运行
在Linux中，env命令用于查看环境变量

环境变量是keyaValue形式即键值对

环境变量 path

  • PATH 是操作系统中用于指定可执行文件搜索路径的核心环境变量，决定命令行能否直接调用程序。
  • path记录了程序的搜索路径，即无论在何处都可以执行记录的程序

$符号

用于直接取环境变量的值

自行设置环境变量

临时生效：语法：export 变量名 = 值
永久生效：

  1. 针对当前用户生效配置在当前用户的~/bashrc文件中
  2. 针对所有用户生效，配置在系统/etc/profile
  3. 通过语法source 配置文件，进行立即生效，或者重新登陆finalshell生效
export PATH=$PATH:hello
vim /etc/profile
#在文件底追写export PATH=$PATH:hello的路径
