权限控制

Linux 查看权限控制信息 核心知识点整理

一、权限核心查看命令

ls -l（简写ll）：列表形式展示文件 / 目录详细信息，可直接获取权限标识、所属用户、所属用户组等权限控制核心信息，是权限查看的基础命令。

二、权限标识解析（10 位字符）

ls -l输出首列的 10 位字符为权限核心标识，按规则分为文件类型+三类用户权限，是理解 Linux 权限的关键：

  1. 第 1 位：文件类型，核心类型为d（目录）、-（普通文件）、l（符号链接）；

  2. 第 2-4 位：所属用户（u）的权限；

  3. 第 5-7 位：所属用户组（g）的权限；

  4. 第 8-10 位：其他用户（o）的权限；

  5. 权限符号：r（读）、w（写）、x（执行），-代表无对应权限。

    r: ls命令

    w: 可以在文件夹里创建，删除，改名

    x:可以cd进入此文件夹

    示例

    ：

    → 目录文件，所属用户有读 / 写 / 执行全权限，用户组和其他用户仅拥有读和执行权限。

三、权限的两种表示方法

Linux 权限有符号表示法和数字表示法两种，可相互转换，适用于不同操作场景：

1. 符号表示法

直接通过r/w/x/-的组合表示权限，对应 u/g/o 三类用户，直观易懂，适合精准分配单类用户权限。

2. 数字表示法（权限值换算）

将符号权限转换为数字，便于快速批量设置权限，是实操中常用方式：

  • 基础换算值：r=4、w=2、x=1、无权限 = 0；

  • 计算规则：单类用户的权限值为对应符号值相加；

  • 常用组合：755（rwxr-xr-x）、644（rw-r–r–）、700（rwx——）。

    示例

    ：

    → 4+0+1=5，

    → 4+2+1=7。

四、权限修改操作（chmod）

1. 核心规则

  • 操作主体：仅文件 / 目录的所属用户或root 用户可修改权限；
  • 递归选项：-R，适用于目录，可对目录内所有文件 / 子目录统一设置权限。

2. 两种修改语法

  • 符号法：chmod (-R) u=权限,g=权限,o=权限 文件名/目录名（精准分配三类用户权限）；
  • 数字法：chmod (-R) 数字权限值 文件名/目录名（高效批量设置，如chmod 755 文件夹名）。

五、所属用户 / 组修改（chown）

作为权限控制的配套核心操作，用于修改文件 / 目录的所属主体：

  1. 操作限制：仅root用户可执行，普通用户无法将文件 / 目录归属修改为其他用户 / 组；

  2. 核心语法：

    • 同时修改用户 + 用户组：chown (-R) 用户名:用户组 文件名/目录名；
    • 单独修改用户：chown (-R) 用户名 文件名/目录名；
    • 单独修改用户组：chown (-R) :用户组 文件名/目录名。

    此处用户名和用户组都是目标用户/用户组

六、Linux 权限核心适用规则

  1. 目录的x权限是访问基础：无x权限时，即便拥有r权限，也无法查看目录内的内容；
  2. 普通文件默认无x权限：系统默认设置，避免误执行未知程序 / 脚本，需执行时手动添加；
  3. 权限匹配优先级：所属用户（u） > 所属用户组（g） > 其他用户（o），系统会按此顺序匹配用户权限。
